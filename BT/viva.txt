1. What is Ethereum?

Ans:
Ethereum is a blockchain platform that supports smart contracts ‚Äî programs that run exactly as coded without any third party.

2. What is Solidity?

Ans:
Solidity is a programming language used to write smart contracts that run on the Ethereum Virtual Machine (EVM).

3. What is Gas in Ethereum?

Ans:
Gas is the unit of computational cost for executing transactions or smart contracts on the blockchain.
You pay gas in ETH as a transaction fee.

4. What is a Wallet Address?

Ans:
It is a unique identifier for each user on the blockchain, used to send or receive Ether and interact with contracts.

5. What is MetaMask used for?

Ans:
MetaMask is a crypto wallet and browser extension that connects your web app or Remix IDE to the Ethereum network and signs transactions.

6. What is a Test Network (Testnet)?

Ans:
A testnet is a network used for testing smart contracts using fake ETH, without spending real money.
Examples: Sepolia, Goerli.

7. What is the difference between Mainnet and Testnet?
Mainnet	Testnet
Uses real Ether (has real value)	Uses fake Ether (no real value)
Used for real-world deployment	Used for testing and learning
Permanent blockchain	Can be reset or changed easily
8. What is EVM (Ethereum Virtual Machine)?

Ans:
EVM is the runtime environment that executes smart contracts on all Ethereum nodes.
It ensures that the same code runs identically everywhere.

9. What is the purpose of require() statement in Solidity?

Ans:
require() is used to check conditions before executing code.
If the condition fails, it reverts the transaction and returns unused gas.

10. What is the difference between view and pure functions?
Type	Meaning	Gas Usage
view	Reads blockchain data but doesn‚Äôt modify it	No gas when called externally
pure	Doesn‚Äôt read or modify blockchain data	No gas when called externally
11. What is mapping in Solidity?

Ans:
A mapping is a key-value store used to associate one type of data with another, e.g.
mapping(address => uint) balances;

12. What is a Struct in Solidity?

Ans:
A struct is a custom data type that groups multiple variables under one name (like a record).
Example: struct Student { uint roll; string name; uint marks; }

13. What is ABI (Application Binary Interface)?

Ans:
ABI defines how data and functions are encoded/decoded so external applications (like MetaMask or Web3.js) can interact with the smart contract.

14. What is a Transaction Hash?

Ans:
It is a unique identifier (like a receipt) for a blockchain transaction, used to track it on explorers like Etherscan.

15. What is the use of Remix IDE?

Ans:
Remix IDE is an online tool for writing, compiling, deploying, and testing smart contracts easily without setting up any local environment.

16. What is the role of Node.js in Smart Contracts?

Ans:
Node.js allows developers to run deployment scripts, interact programmatically with contracts, and connect blockchain APIs using libraries like Web3.js or Ethers.js.

17. What happens when a transaction fails in Ethereum?

Ans:
If a transaction fails (like due to failed require()), the state is reverted, but gas fee is still consumed because computation was attempted.

18. What is a Faucet in blockchain?

Ans:
A faucet is a website that gives free test Ether for trying smart contracts on test networks.

19. What is the use of public and private in Solidity?

Ans:
They are visibility specifiers that control whether a variable or function can be accessed outside the contract.

20. What is the importance of Gas Fee Observation in BT-2?

Ans:
Observing gas helps understand how different operations consume computational resources, helping in optimizing smart contracts for real deployment.

### **1. What is a Smart Contract?**

**Definition:**
A **smart contract** is a **self-executing program** stored on a **blockchain** that automatically executes and enforces rules or agreements when certain predefined conditions are met.

**Detailed Explanation:**

* It is similar to a **digital agreement** written in code instead of paper.
* Once deployed on a blockchain (like Ethereum), it **runs automatically** without any third-party interference.
* The terms of the agreement are directly written into lines of code.
* It ensures **trust**, **transparency**, and **security** because once deployed, the code **cannot be altered**.

**Example:**
Let‚Äôs say you write a smart contract for selling an NFT.

* Condition: When a buyer sends 1 ETH,
* Action: The smart contract automatically transfers the NFT to the buyer.

You don‚Äôt need a middleman ‚Äî the blockchain handles it.

**Key Features:**

* **Immutable:** Once deployed, code cannot be changed.
* **Transparent:** Everyone can verify the code and transaction.
* **Autonomous:** Executes automatically when conditions are met.
* **Secure:** Based on cryptographic blockchain security.

### **2. Explain use of MetaMask for Smart Contracts.**

**Definition:**
**MetaMask** is a **crypto wallet** and **gateway to blockchain applications**.
It allows users to **interact with Ethereum-based smart contracts** directly from their browser.

**Use in Smart Contracts:**

* MetaMask connects your **web application (frontend)** with the **Ethereum blockchain**.
* It manages **your Ethereum account**, **private keys**, and **transactions** securely.
* When deploying or interacting with a smart contract, MetaMask:

  1. **Signs** transactions using your wallet address.
  2. **Sends** transactions to blockchain nodes.
  3. **Pays** gas fees automatically.

**Steps for Use:**

1. Install MetaMask (browser extension or mobile app).
2. Create or import a wallet.
3. Connect MetaMask with your dApp (Decentralized App).
4. When the app calls a smart contract, MetaMask prompts for approval.
5. After approval, MetaMask broadcasts the transaction to the blockchain network (mainnet or testnet).

**Example:**
When you click ‚ÄúDeploy Smart Contract‚Äù in your app, MetaMask pops up asking for confirmation and gas fee ‚Üí once confirmed, the contract is deployed to blockchain.

### **3. Write down steps for creating Node application for smart contracts deployment.**

Here we are using **Node.js + Hardhat** or **Truffle** for smart contract deployment.

**Steps (Generalized Process):**

1. **Install Node.js and npm**

   * Node.js helps to run JavaScript outside browser.
   * `npm` (Node Package Manager) installs required blockchain tools.

   ```bash
   node -v
   npm -v
   ```

2. **Create Project Folder**

   ```bash
   mkdir smart-contract-demo
   cd smart-contract-demo
   ```

3. **Initialize Node Project**

   ```bash
   npm init -y
   ```

4. **Install Hardhat or Truffle**

   * Hardhat is a popular Ethereum development framework.

   ```bash
   npm install --save-dev hardhat
   npx hardhat
   ```

5. **Write Smart Contract**

   * Create a file in `/contracts` folder (e.g., `SimpleStorage.sol`)
   * Example:

     ```solidity
     // SPDX-License-Identifier: MIT
     pragma solidity ^0.8.0;

     contract SimpleStorage {
         uint public data;
         function set(uint _data) public {
             data = _data;
         }
     }
     ```

6. **Compile the Contract**

   ```bash
   npx hardhat compile
   ```

7. **Deploy Script**

   * Create a deploy script in `/scripts/deploy.js`

   ```javascript
   const hre = require("hardhat");
   async function main() {
     const Storage = await hre.ethers.getContractFactory("SimpleStorage");
     const storage = await Storage.deploy();
     await storage.waitForDeployment();
     console.log("Contract deployed at:", await storage.getAddress());
   }
   main();
   ```

8. **Run Deployment on Test Network**

   ```bash
   npx hardhat run scripts/deploy.js --network sepolia
   ```

9. **Connect MetaMask**

   * Use same test network (e.g., Sepolia) and wallet address from MetaMask to view deployed contract.

### **4. Explain concept of Test Network and provide its types.**

**Definition:**
A **test network (testnet)** is a **blockchain network used for testing** smart contracts or decentralized apps before deploying them to the **main network (mainnet)**.

**Purpose:**

* To experiment with contracts **without spending real money**.
* To **debug and verify** logic before real-world deployment.
* To simulate how transactions will behave on mainnet.

**How it Works:**

* Testnets use **test tokens** (e.g., test ETH) that have **no real-world value**.
* You can get free test tokens from **faucets**.
* The structure and rules are the same as the mainnet, but separate.

**Types of Ethereum Test Networks:**

1. **Sepolia Testnet** ‚Üí Modern official Ethereum testnet (widely used).
2. **Goerli Testnet** ‚Üí Previously popular, now deprecated in 2025.
3. **Rinkeby Testnet** ‚Üí Older network, replaced by Sepolia.
4. **Local Testnet** ‚Üí Created using tools like Hardhat or Ganache for local testing.

**Example:**
If you want to test your contract deployment, you can deploy it on **Sepolia** using fake ETH from a faucet ‚Üí verify the contract on Etherscan testnet ‚Üí then deploy on **Ethereum Mainnet** once confirmed working.
